# Energy Minimization (EM) in AMBER
![AmberTools 2025](https://img.shields.io/badge/AmberTools-2025-blue)
![Amber2025](https://img.shields.io/badge/Amber-2025-red)

Energy Minimization (EM) is the process of finding a structure of a molecular system that corresponds to a local minimum on its potential energy surface (PES). This is achieved by iteratively adjusting the positions of the atoms until the force on each atom is approximately zero, resulting in an **"optimized geometry"** or **"minimum energy structure"**.

EM serves as a critical first step in Molecular Dynamics (MD) simulations to eliminate steric clashes and prepare the system for subsequent equilibrium phases. 

---

## 📋 Methodology Overview

The minimization is performed **"in vacuum"** (without explicit thermodynamic parameters indicating temperature, volume, or pressure) across **four sequential steps** with gradually decreasing harmonic constraints on the solute:

1. **`em.in`**: Tight constraints at **20.0 kcal/mol**
2. **`em2.in`**: Moderate constraints at **10.0 kcal/mol**
3. **`em3.in`**: Light constraints at **5.0 kcal/mol**
4. **`em4.in`**: Unconstrained minimization (**0.0 kcal/mol**)

> ⚠️ **Important:** Minimizations must be run sequentially. From the second minimization onwards, the coordinate (`-c`) and reference (`-ref`) flags must point to the restart file (`restrt`) generated by the immediate preceding step.

---

## 🛠️ Input File Setup (`.in`)

Four separate parameter files are required. Below are the configurations to be included in each file.

### Required Custom Parameters
Ensure the following parameters are added to your input blocks:
* `cut = 9.0` — Non-bonded interactions cutoff distance.
* `maxcyc = 2000` — Total number of minimization cycles.
* `ncyc = 500` — Performs 500 steps of *Steepest Descent* minimization, followed by 1500 steps (`maxcyc - ncyc`) of *Conjugate Gradient* minimization.
* `ntb = 1` — Periodic Boundary Conditions (PBC) for constant volume.
* `ntpr = 1` — Print the progress of the minimization every `ntpr` step.
* `iwrap = 1` — Enforcement of PBC atom wrapping conditions.
* `ntwx = 100` — Frequency of writing coordinates to the trajectory file (see **Important Note** below).

### Restraint Specification Syntax
Restraints are specified at the end of the input file after the closing slash (`/`) using the following wording:

```text
Hold the Ubiquitin fixed 20.0
RES 1 76
END
END
```

Once the files are prepared, the pmemd command is invoked to run the minimization. Example: 

```bash
pmemd -O -i em.in -p complex.prmtop -c complex.rst7 –ref complex.rst7 -o mdout.em1 -inf mdinfo.em1 -r restrt.em1 -x mdcrd.em1
```

where *mdinfo* is the file with the run specifications, while *mdout* is the file where you can check how the run is proceeding (you can see the time and the various parameters of the specific step you are checking). The *mdcrd* file indicates the trajectory file.


# 
Energy Minimization is a purely static optimization method that focuses exclusively on mapping forces down to zero to find the most stable geometric conformation. It entirely neglects temporal evolution, particle velocities, and kinetic energy.
* Why is the mdcrd file empty or unusable? Despite explicitly specifying a value of ntwx=100 in the .in file, the generated mdcrd file will not contain active frames describing the system's evolution over time.
* How to get a real trajectory? A time-dependent trajectory is only generated and populated during Dynamic Simulations (Heating, Equilibration, and Production phases), where molecular motion is governed by classical Newtonian mechanics over actual time steps.

In summary, EM is a static process designed to establish a stable structural baseline. To observe and analyze the movement of molecules over time, you must complete the minimization sequence and proceed with dynamic molecular dynamics steps.
